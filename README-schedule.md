# AI日程规划功能 - 实现说明

## 功能概述

AI日程规划功能允许用户通过点击按钮，调用后端AI接口获取智能规划的日程事件，生成标准的.ics日历文件并触发浏览器下载。

## 技术栈

- **前端框架**: Vue 3 + Vite
- **日历生成**: ics库
- **API调用**: Fetch API
- **样式**: CSS3 + Flexbox/Grid

## 功能特性

### 1. 界面元素

- **触发按钮**: "📅 生成日程并添加到日历"
- **加载状态**: 点击后显示"AI正在为您规划日程..."和加载动画
- **状态提示**: 成功/错误消息提示
- **Mock切换**: 开发模式下可切换使用Mock数据

### 2. 交互逻辑

1. **收集请求参数**:
   - query: 用户原始输入或商品描述
   - context_time: 当前时间的ISO字符串
   - extra_info: 店铺名称、地址等额外信息

2. **调用后端接口**:
   - 接口地址: POST /api/generate_schedule
   - 处理响应成功/失败状态

3. **生成.ics日历文件**:
   - 使用ics库将事件转换为标准日历格式
   - 处理时间格式转换（ISO → 数组）

4. **触发浏览器下载**:
   - 使用Blob创建text/calendar类型文件
   - 创建隐藏的<a>标签模拟点击下载

### 3. Mock备用方案

- 内置静态Mock事件数据
- 通过环境变量`VITE_USE_MOCK`控制
- 当后端不可用时自动降级使用Mock数据

### 4. 异常处理

- 网络错误处理
- 后端返回格式错误处理
- ics生成失败处理
- 多层降级机制

## 使用方法

### 开发环境

1. **安装依赖**:

   ```bash
   npm install
   ```

2. **启动开发服务器**:

   ```bash
   # 仅启动前端
   npm run dev

   # 同时启动前端和Mock服务器
   npm run dev:full

   # 单独启动Mock服务器
   npm run mock:server
   ```

3. **访问应用**:
   - 前端: http://localhost:5173
   - Mock API: http://localhost:3001

### 生产环境

1. **构建应用**:

   ```bash
   npm run build
   ```

2. **预览构建结果**:
   ```bash
   npm run preview
   ```

## API接口说明

### 请求格式

```json
POST /api/generate_schedule
{
  "query": "我想吃披萨，选了必胜客超级至尊披萨",
  "context_time": "2026-04-18T17:00:00.000Z",
  "extra_info": {
    "shop_name": "必胜客中关村店",
    "address": "中关村大街1号"
  }
}
```

### 响应格式

```json
{
  "success": true,
  "message": "AI已成功规划日程",
  "events": [
    {
      "title": "AI规划：披萨晚餐",
      "start": "2026-04-18T19:00:00.000Z",
      "end": "2026-04-18T20:30:00.000Z",
      "location": "必胜客中关村店",
      "description": "根据您的需求智能规划的日程"
    }
  ]
}
```

## 测试要点

## 注意事项

1. **时区处理**: ics库使用本地时间，确保时间转换正确
2. **文件命名**: 下载的文件名为`ai_schedule.ics`，可根据需求修改
3. **API安全**: 生产环境需要添加API密钥验证和限流
4. **错误处理**: 确保所有可能的错误都有相应的处理逻辑

## 故障排除

### 常见问题

1. **无法下载文件**:
   - 检查浏览器是否阻止弹出窗口
   - 检查Blob API是否支持

2. **时间显示错误**:
   - 检查ISO时间转换函数
   - 验证时区设置

3. **API调用失败**:
   - 检查网络连接
   - 验证API地址和CORS配置
   - 查看浏览器控制台错误信息

### 调试技巧

1. 使用浏览器开发者工具查看网络请求
2. 检查控制台日志输出
3. 使用Mock模式隔离前端问题
4. 测试.ics文件在日历应用中的兼容性
